    注册、登录、用户管理¶

    概述¶

    用户管理是网关重要的功能之一。

    如果当前的用户 id 在网关中并没有记录的情况，就需要进注册的步骤。这里有两种情况，一是网关中没有管理员，此时注册的话，当前注册的用户 id 即会成为网关的管理员；二是网关已经有管理员，当前注册的用户 id 成为普通用户，这个注册的请求需要管理员去授权。通过注册和登录的接口都能知道当前用户 id 是否已注册、是否已被授权。

    注册¶

    • Android
    HostApi.register(hostId, "123456", new IBaseCallback<HostRegister.Recv>() {
    @Override
    public void onSuccess(HostRegister.Recv recv, int callBackMark, String errorCode) {
        if ("10017".equals(errorCode)) {
            // 等待网关管理员同意"
        } else if ("10001".equals(errorCode)) {
            String sspen = recv.getSspen();
            if ("1".equals(sspen)) {
                // 已注册网关, 并且已授权
            } else {
                // 已注册网关, 但管理员未授权
            }
        } else {
            // 注册网关成功
        }
    }

    @Override
    public void onFailure(int callBackMark, String errorCode) {
        if ("10026".equals(errorCode)) {
            // 网关标识错误
        } else {
            // 注册网关失败
        }
    }
});

    登录¶

    • Android
    // 登录网关
HostApi.login(hostId, "123456", new IBaseCallback<HostLogin.Recv>() {
    @Override
    public void onSuccess(HostLogin.Recv recv, int callBackMark, String errorCode) {
        VensiMqtt.getInstance().setMainHost(recv.getHostId());
        // 登录网关成功
    }

    @Override
    public void onFailure(int callBackMark, String errorCode) {
        if ("10002".equals(errorCode)) {
            // 密码错误
        } else if ("10025".equals(errorCode)) {
            // 等待网关管理员同意
        } else {
            // 注册网关失败
        }
    }
});

    用户管理¶

    监听授权管理¶

    通过下面设置回调，可以监听授权申请、授权结果、是否成为了管理员

    • Android
    ReportMsgApi.setOnHostReport(new ReportMsgApi.OnHostReport() {

    // ...

    @Override
    public void onAuthorizationResult(String hostId, HostAuthorizationResult result) {
        // 网关授权结果
        switch(result){
            case HostAuthorizationResult.agree:
                // 管理员同意
                break;
            case HostAuthorizationResult.reject:
                // 管理员拒绝
                break;
            case HostAuthorizationResult.delete:
                // 被管理员删除
                break;
        }
    }

    @Override
    public void onApplyAuthorization(String hostId, String userId) {
        // 有人申请网关的权限（当前是管理员的情况下才有上报）

        // 此时可以跳转到用户管理界面, 调用 HostApi.getUserList(String hostId) 获取用户列表
        // 获取回来的 HostUserList.Recv.User 对象。
        // 其中 userRights 参数为当前用户的级别，如果为“admin”则为管理员，“guest”则为普通用户
        // 在普通用户中，sspen 参数代表该用户是否需要授权，“0”为等待授权，其他则为已经授权。

        // 调用 HostApi.dealAuthorization(String hostId, String authUserId, boolean isRefuse) 进行授权操作
        // 因为网关设计的问题，拒绝授权的话，直接调用删除用户接口 HostApi.deleteUser(String hostId, String removeUserId)
    }

    @Override
    public void onTransferAdmin(String hostId) {
        // 已成为网关管理员
    }
});
