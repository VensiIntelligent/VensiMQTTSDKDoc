    自动化¶

    网关的自动化功能包括了三个方面。

    • 定时自动化。在一天的某个时间执行某个操作。例如每天早上八点打开客厅的窗帘。
    • 设备触发自动化。例如开关一打开了，触发开关二也一起打开。
    • 传感器状态变化触发。例如门磁传感器检测到门打开了，触发门廊灯打开。

    创建、编辑自动化¶

    创建自动化的步骤

    • 1、创建自动化触发的条件，我们称之为触发器。
    • 2、选择需要执行的设备和情景（可以只有设备或者只有情景或者两者都有，而且她们的数量不受限制）

    创建触发器¶

    • 1、定时的自动化，只需要创建一个 Action

    • Android

    // 创建时间点 例如 12:00
Calendar calendar = Calendar.getInstance();
calendar.set(Calendar.HOUR_OF_DAY, 12);
calendar.set(Calendar.MINUTE, 0);
Date time = calendar.getTime();

AutoApi.ConditionTimeActionBuilder conditionTimeActionBuilder = new AutoApi.ConditionTimeActionBuilder()
    .setName("每天 12：00 触发动作") // 动作名称
    .setDescription("") // 动作释义
    .setTime(time) // 时间点
    .setWeekdays(Weekdays.values()); // 星期周期 Weekdays.MONDAY Weekdays.TUESDAY 等，这里可以填入一个或多个

List<AutoApi.ConditionActionBuilder> conditionActionBuilders = new ArrayList<>();
conditionActionBuilders.add(conditionTimeActionBuilder);

AutoApi.ConditionBuilder conditionBuilder = new AutoApi.ConditionBuilder()
    .setName("每天 12：00 触发条件") // 触发器名称
    .setDescription("每天 12：00 触发条件") // 触发器释义
    .setCreateDate(new Date()) // 触发器的创建时间
    .setAction(conditionActionBuilders); // 将上面创建的动作传入

    • 2、设备触发自动化、传感器触发自动化，需要创建一个时间 Action，一个设备 Action

    • Android

    // 时间 Action
AutoApi.ConditionTimeActionBuilder conditionTimeActionBuilder = new AutoApi.ConditionTimeActionBuilder()
    .setName("星期一到星期五 08:30-18:30") // 动作名称
    .setDescription("工作日") // 动作释义
    .setTime(startTime, endTime) // 时间段，如果是某个时间就调用 setTime(Date)
    .setWeekdays(Weekdays.MONDAY, Weekdays.TUESDAY, Weekdays.WEDNESDAY, Weekdays.THURSDAY, Weekdays.FRIDAY); // 星期

// 设备 Action
AutoApi.ConditionDeviceActionBuilder triggerDeviceActionBuilder = new AutoApi.ConditionDeviceActionBuilder()
    .setDeviceId(device.getDeviceId()) // 设备 id
    .setDeviceName(device.getDeviceName()) // 设备名称
    .setDeviceType(device.getDeviceType() + device.getZoneId()) // 设备类型
    .setKey("onoff") // 触发时的状态参数
    .setValue("on")  // 触发时的值
    .setDescription("当" + device.getDeviceName() + "打开的时候"); // 释义

List<AutoApi.ConditionActionBuilder> conditionActionBuilders = new ArrayList<>();
conditionActionBuilders.add(conditionTimeActionBuilder);
conditionActionBuilders.add(triggerDeviceActionBuilder);

AutoApi.ConditionBuilder conditionBuilder = new AutoApi.ConditionBuilder()
    .setName("星期一到星期五 08:30-18:30 当[" + device.getDeviceName() + "]打开的时候") // 触发器的名称
    .setDescription("星期一到星期五 08:30-18:30 当[" + device.getDeviceName() + "]打开的时候") // 触发器的释义
    .setCreateDate(new Date()) // 触发器的创建时间
    .setAction(conditionActionBuilders); // 将上面创建的时间 Action 和设备 Action 闯入

    创建执行动作¶

    在上面设置触发器成功后，开始设置自动化的执行动作参数。

    执行动作可以是设备（或多个设备）和情景（或多个情景），或者设备和情景都有，但至少需要有一个设备或一个情景

    • Android
    // 如果是编辑自动化，就保持自动化的 id 不变 AutoApi.AutoBuilder(autoId)
AutoApi.AutoBuilder autoBuilder = new AutoApi.AutoBuilder();

// 将上面步骤创建的触发器闯入
autoBuilder.setCondition(conditionBuilder);

// 执行的设备（可以一个或多个）
ArrayList<AutoApi.AutoActionBuilder> actionBuilders = new ArrayList<>();
AutoApi.AutoDeviceActionBuilder autoDeviceActionBuilder = new AutoApi.AutoDeviceActionBuilder()
    .setDeviceId(device.getDeviceId()) // 设备 id
    .setDeviceType(device.getDeviceType() + device.getZoneId()) // 设备类型
    .setKey("status") // 执行的参数名
    .setValue("on")   // 执行的值
    .setOrderIndex("0") // 有多个设备的时候，可以进行排序
    .setDelayTime("0"); // 有多个设备的时候，可以设置他们之前的间隔
actionBuilders.add(autoDeviceActionBuilder);
autoBuilder.setActions(actionBuilders);

// 执行的情景（可以一个或多个）
ArrayList<AutoApi.AutoSceneBuilder> autoSceneBuilders = new ArrayList<>();
AutoApi.AutoSceneBuilder autoSceneBuilder = new AutoApi.AutoSceneBuilder(scene.getId())
    .setName(scene.getName()) // 情景的名称
    .setImg("0") // 情景的图标类型（自行定义）
    .setOrderIndex("0"); // 有多个情景的时候，可以进行排序
autoSceneBuilders.add(autoSceneBuilder);
autoBuilder.setScenes(autoSceneBuilders);

autoBuilder.setName("自动化") // 自动化名称
    .enablePushMessage(false) // 消息是否开启推送，推送是走的推送厂商（例如极光推送，这个需要联系开发获取更多功能）
    .setType(AutoType.device) // 自动化的类型 time定时 device设备 linkage传感器
    .setCreateDate(new Date()); // 创建时间

AutoApi.edit(hostId, autoBuilder, new MqttCallback<>());

    删除自动化¶

    • Android
    AutoApi.delete(hostId, auto.getId(), new MqttCallback<String>() {

    @Override
    public void onSuccess(String s, int callBackMark, String errorCode) {
        // 删除成功
    }

    @Override
    public void onFailure(int callBackMark, String errorCode) {
        // 删除失败
    }
});

    启用，关闭自动化¶

    关闭自动化之后，该自动化不会再触发。

    • Android
    boolean value = true; // true 启用 false 关闭
AutoApi.editAutoStatus(hostId, auto.getId(), true, new MqttCallback<String>() {

    @Override
    public void onSuccess(String s, int callBackMark, String errorCode) {
        // 设置成功
    }

    @Override
    public void onFailure(int callBackMark, String errorCode) {
        // 设置失败
    }
});
