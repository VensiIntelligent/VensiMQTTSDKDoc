    设备控制，状态监听¶

    设备控制¶

    一般的 zigbee 设备我们通过一下接口进行状态的控制，其他特殊的设备控制请查看**特殊设备说明**

    • Android
    // 控制某个设备的状态
// 查看下面附录一查看不同设备对应的 controlKey 和 controlValue 值
DeviceApi.control(String hostId, String deviceId, String deviceType, String zoneId, String controlKey, String controlValue, IBaseCallback<String> callback)

    状态监听¶

    • Android
    // 主动获取设备状态
// 获取多个设备状态
List<DeviceState.Publish.DeviceInfo> deviceInfoList = new ArrayList<>();
for (DeviceList.Recv.Device device : mDevices) {
    DeviceState.Publish.DeviceInfo info = new DeviceState.Publish.DeviceInfo();
    info.setId(device.getDeviceId());
    info.setDeviceType(device.getDeviceType());
    info.setZoneId(device.getZoneId());
    deviceInfoList.add(info);
}
DeviceApi.getStatus(hostId, deviceInfoList, new MqttCallback<DeviceState.Recv>() {
    @Override
    public void onSuccess(DeviceState.Recv recv, int callBackMark, String errorCode) {
        // 多个设备的状态
        recv.getStateList();
    }
});

// 获取某个设备状态
DeviceApi.getStatus(hostId, deviceId, deviceType, zoneId, new MqttCallback<DeviceState.Recv>() {
    @Override
    public void onSuccess(DeviceState.Recv recv, int callBackMark, String errorCode) {
        // 某个设备的状态
        recv.getStateList().get(0);
    }
});


// 状态监听
ReportMsgApi.setOnDeviceReport(new ReportMsgApi.OnDeviceReport() {

    // ...

    @Override
    public void onStatusReport(DeviceStateRecv recv) {
        // 设备状态上报
    }

    @Override
    public void onSensorStatusReport(DeviceSensorReportRecv recv) {
        // 传感器状态上报
    }  

    @Override
    public void onSensorStatusReport(DeviceSensorReportRecv recv) {
        // 获取多个设备状态, 当获取不同模块的设备状态时，网关会分几次上报，这个时候就需要用该接口了
    }   
});

    附录一 设备对应的控制值¶

    设备名称 控制参数 控制值
    开关 onoff on/off
    调光灯 onoff on/off
    0~100:亮度
    彩灯 onoff on/off
    0~100:亮度
    254#254#254(hsl, 值范围是 0 ~ 254), 注意, 最后⼀个 l 是⽆效的)
    ⾊温灯 onoff on/off
    ⾊温灯 color_temp 默认是0~500，某些设备会上报自己的色温范围，已上报的为准
    插座 onoff on/off
    窗帘开关 onoff open/close/stop
    窗帘电机 onoff 进度0~100
    电源控制器 onoff on/off
    晾⾐机 motor 移动电机位置 rising/falling/stop
    晾⾐机 lighting 照明 on/off
    晾⾐机 heating 烘⼲ on/off
    晾⾐机 heating_time 烘⼲时间 1~180(min)
    晾⾐机 wind ⻛⼲ on/off
    晾⾐机 wind_time ⻛⼲时间 1~180(min)
    晾⾐机 disinfection 消毒 on/off
    晾⾐机 disinfection_time 消毒时间 1~30(min)
    声光报警器 onoff on: 打开默认报警模式
    off: 关闭报警
    strobe: 闪灯报警
    siren: 声⾳报警
    strobe-siren:声光报警
    door:破⻔报警
    fire:⽕警
    sos:sos紧急报警
    leak:漏⽔报警
    gas:烟雾报警
    声光报警器 warn_mode 报警模式
    strobe:闪灯报警
    siren:声⾳报警
    strobe-siren:声光报警
    声光报警器 duration 报警持续时⻓ 1~65535 默认240 单位s
    声光报警器 duty_cycle 闪灯占空⽐ 1~100 默认50 值越小，红灯持续时间约短
    声光报警器 siren_level 声⾳报警等级 0~3
    ⽑⼱架 onoff 1:表明是关机
    2:表明是待机

    4:表明是智能模式定时2⼩时

    6:表明是智能模式定时4⼩时
    7:表明是消毒模式定时30分钟
    8:表明是消毒模式定时45分钟
    9:表明是消毒模式定时60分钟
    10:表明是烘⼲模式定时1⼩时
    11:表明是烘⼲模式定时2⼩时
    12:表明是烘⼲模式定时3⼩时
    13:表明是烘⼲模式定时4⼩时
    14:表明是送⻛模式定时1⼩时
    15:表明是送⻛模式定时2⼩时
    16:表明是送⻛模式定时3⼩时
    17:表明是送⻛模式定时4⼩时
    ⽑⼱架 heating 加热 0:关闭/1开启

    0:⼿动模式

    3:极速模式

    1:⼆级

    空⽓净化器 anion 负离⼦ on/off
    空⽓净化器 child_lock 童锁 on/off
    空⽓净化器 work_timing 定时运⾏ 0~12 单位小时 0为定时取消

    附录二 设备对应状态上报值¶

    没有特殊说明，设备的控制参数和上报的参数是一致的，例如开关，控制是 onoff，上报也是 onoff

    设备名称 控制参数 控制值
    调光灯 lumin 亮度:0~100
    彩灯 hue ⾊相:0~100
    彩灯 saturate 饱和度:0~100
    彩灯 lumin 亮度:0~100
    晾⾐机 motor_pos 当前电机位置位置
    top:顶部
    bottom:底部
    middle:中间
    ⽑⼱架 temp 温度 -50~90摄氏度
    ⽑⼱架 model_timer 模式剩余时间
    空⽓净化器 pm_25 0-500
    空⽓净化器 run_time 累计运⾏时间 单位小时
    空⽓净化器 count_down 定时倒计时 单位分钟
    空⽓净化器 filter_time 滤芯1已使⽤时间 单位分钟
    净⽔器 inTDS 原⽔TDS(溶解性固体总量) 单位ppm
    净⽔器 outTDS 出⽔TDS(溶解性固体总量) 单位ppm
    净⽔器 totalML 总净⽔量 单位ml
    净⽔器 filter_pct 滤芯1寿命 单位%
    净⽔器 filter_time 滤芯1使⽤时间 单位小时
    净⽔器 filter2_pct 滤芯2寿命 单位%
    净⽔器 filter2_time 滤芯2使⽤时间 单位小时
