    打开或关闭红外学习¶

    返回结果需要在ReportMsgApi.setOnInfraredReport(ReportMsgApi.OnInfraredReport)中监听

    public static long studyInfrared(String hostId, String deviceId, String subDeviceId, String keyId, String keyName, boolean isOpen, final IBaseCallback<InfraredForwarderLearing.Recv> callback)
    参数 释义
    hostId 网关id
    deviceId 设备id
    subDeviceId 子设备Id
    keyId 按键Id： 传⼊⾃⼰创建的 UUID 即可(注意, ⻓度固定16)
    keyName 按键名称
    isOpen close_study: 关闭红外学习 open_study: 打开红外学习(30秒⾃动关闭)
    callback 回调

    public class InfraredForwarderLearnSettingKeyNamePublish {
  @SerializedName("device_id")
  private String deviceId; // 设备id
  @SerializedName("type_id")
  private String typeId; // 类型id

  @SerializedName("key_id")
  private String keyId; // 按键id
  @SerializedName("key_name")
  private String keyName; // 按键名称

  @SerializedName("img_id")
  private String imgId;  // 图标id
  @SerializedName("key_seq")
  private String keySeq; // 排序序号
}

    获取某个类型子设备的配置步骤(云端接口)¶

    获取不同类型子设备的测试步骤，这样能准确定位是不是该类型的子设备

    比如获取电视的配置步骤，开关，音量，菜单 这几个步骤。 然后就一个个试过去. 全都有响应才能准确定位是该类型的子设备

    public static long getSubDeviceConfigStep(String hostId, String typeId, String brandId, String remoteId, IBaseCallback<InfraredForwarderFit.Recv> callback)
    参数 释义
    hostId 网关id
    typeId 类型id
    brandId 品牌id
    remoteId 型号id
    callback 回调
